Anonymous FTP Setup

Your anonymous ftp site is completely different from your web site.

When people ftp to your domain anonymously, they will see the following directories:

bin/ dev/ etc/ incoming/ lib/ pub/

"pub" is where you should put all your anonymously accessible files.
"incoming" is for the anonymous users to upload files.

For security the following applies:

  • Only the incoming directory can be written to anonymously
  • Subdirectories are not creatable
  • The incoming directory is not readable by people dropping files there

You are responsible for any "pirated" software uploaded by the anonymous users. The anonymous ftp sites will be periodically monitored for any abuses.

You may ignore the other directories.

To access the anonymous FTP site from the Web, use the following address:

ftp://yourdomain.com/pub/

Your HTML to download a file called mirc511s.exe from a webpage would look like this:

<A HREF="ftp://yourdomain.com/pub/mirc511s.exe">Download Mirc Now</A>

You should tell your visitors that they may need to right-click on the link if they are Windows users, or if they use Mac OS, that they may need to Control-click or hold down the mouse button on the link, then select the appropriate option from the pop-up menu.

Please note that it's possible during peak hours to receive a "too many anonymous users error", We must restrict the number of simultaneous anonymous users to keep the webserver and normal FTP performance within normal limits. If you receive this error often you may want to put you downloadable files in your main web directory and link to them with an http call.

Example:

http://yourdomain.com/files.zip

This assumes the zip files are in the main web directory.

Note - if you experience slow download performance chances are it's a problem on the internet. We maintain a large margin of available bandwidth so bottlenecks never occur within our network. Please feel free to send an inquiry to be sure all the servers are operating properly, if you include your ISP's IP# we will run traceroutes to see where the delays are occuring.

Setting up Your Own "404:File Not Found" Error Page

A file already exists in the www directory of your server. It's called missing.html. You can edit it to your liking, or create your own. As long as it's called missing.html and it's in your root www directory, the server will display it whenever someone tries to access a page on your domain that does not exist.

Password-Protecting Directories

NOTE: Frontpage users do not use this information for it will corrupt your Frontpage Extensions.

To password protect your directories you will need to do two things:

  • Make a list of usernames and passwords.
  • Make a .htaccess file.

Making a List of Usernames and Passwords

To make a list of usernames and passwords you will need to make a file where you can store this information.

Suppose you want to make a file called users located in a directory called protected in your www directory, while adding the username "john" with the password "doe".

To accomplish this you will need to telnet into your account and type:

htpasswd -c /home/username/www/protected/users john

After entering the above code, it will automatically prompt you to enter and then re-enter a password for the username "john".

Now let's look at the above command and see exactly what we did.

  • htpasswd - this is the command used to create the username and password. You can also use it to modify the password of a current user. You must use htpasswd to create your usernames and passwords.
  • -c - This is the command that tells the computer to create a new file. You only need to use this command the first time.

    NOTE - If you use this command after you already have a file created, it will copy over the old file with the new blank file erasing all your previous usernames and passwords.

  • /home/username/www/protected/users - This tells the computer where to make and add the new username.
  • john - This sets the username to john. Just change john to whatever you want when making your own username and password list.

If you were to download your users file and view it in a text editor it would look something like this:

john:8nP9061qRS5OD

Notice the password (8nP9061qRS5OD). It is encrypted by htpasswd, That is why you must use htpasswd to enter your usernames and passwords. Your users file would look something like this after adding a few more usernames and passwords:

john:8nP9061qRS5OD
tom:WrU808BHQai36
richard:iABCQFQs40E8M
harry:FAdHN3W753sSU

That completes the first step in password protecting your directories.

Creating Your .htaccess File

Now that you have set up a database of usernames and passwords you need to apply them to an actual directory. To do this you must create your .htaccess file and upload it to the directory you want password protected.

The best way we have found to make a .htaccess file is to create it in a text editor and save it as .htaccess.txt. Then once you upload it you can rename it on the server with your FTP program to .htaccess (no extension).

NOTE - You must upload and download your .htaccess in ASCII mode (not Binary).

You will need to have the following lines of code in your .htaccess file:

AuthName "Restricted Stuff"
AuthType Basic
AuthUserFile /home/username/www/protected/users

require valid-user

Now lets go over what that code does:

  • AuthName "Restricted Stuff" - This is just the name of your .htaccess file. It could be whatever you like but in this example we used Restricted Stuff.
  • AuthType Basic - This sets the type of password encryption (currently, Basic is the only type available but soon you will be able to choose from Basic or Digest).
  • AuthUserFile /home/username/www/protected/users - This tells your browser where to find the file you made previously that stores all your usernames and passwords.
  • require valid-user - This tells the browser to give access to anyone that enters a valid username and password from your users file. You can narrow this down and only give a select few users from your users file access by modifying the command (i.e. require valid-user john harry. that code would only give access to john and harry).

After you create, edit, and save your .htaccess file, you're ready to upload it. The location of your .htaccess file determines what directory will require a username and password to access it. For example, if you upload it to your www directory, your www directory will require a username and password to access it.
